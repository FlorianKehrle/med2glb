## 4. MATERIAL CONVERSION: GLTFAST → STANDARD (ConvertGltfMaterialsToStandard, lines 320-510)

**When:** Runtime, immediately after model instantiation
**What Gets Converted:** All glTFast/Unlit → Standard shader
**Exception:** Metadata objects (legend/info) retain glTFast shaders

**Property Mapping:**

1. **Base Color** (lines 360-373)
   - glTF property: baseColorFactor → Standard: _Color
   - glTF property: baseColorTexture → Standard: _MainTex
   - Tiling/offset preserved

2. **Metallic & Roughness** (lines 375-387)
   - glTF: metallicFactor → Standard: _Metallic
   - glTF: roughnessFactor → Standard: _Glossiness (inverted: 1 - roughness)
   - **NOTE:** Texture channel layout differs; only scalar values transferred

3. **Normal Maps** (lines 389-400)
   - glTF: normalTexture → Standard: _BumpMap
   - Enabled if glTF material has _NORMALMAP keyword enabled

4. **Occlusion** (lines 402-412)
   - glTF: occlusionTexture → Standard: _OcclusionMap
   - Requires _OCCLUSION keyword in glTF material

5. **Emission** (lines 414-448)
   - glTF: emissiveFactor → Standard: _EmissionColor
   - glTF: emissiveTexture → Standard: _EmissionMap
   - For Unlit materials: simulates unlit by using full emission

6. **Transparency/Alpha** (lines 450-488)
   - glTF _ALPHABLEND_ON → Standard Mode 3 (Transparent/Premultiply)
   - glTF _ALPHAPREMULTIPLY_ON → Standard Mode 3
   - glTF _ALPHATEST_ON → Standard Mode 1 (Cutout)
   - Detects implicit transparency from base color alpha < 1.0

7. **Double-Sided** (lines 490-496)
   - glTF _Cull property (0 = Off) → Standard _Cull property

8. **Shared Material Caching** (lines 329-353)
   - Reuses converted materials across renderers to avoid duplication

**Example Conversion Code:**
`csharp
Material stdMat = new Material(standardShader);
stdMat.name = mat.name;
if (mat.HasProperty("baseColorFactor"))
    stdMat.SetColor("_Color", mat.GetColor("baseColorFactor"));
if (mat.HasProperty("baseColorTexture"))
{
    var tex = mat.GetTexture("baseColorTexture");
    if (tex != null)
    {
        stdMat.SetTexture("_MainTex", tex);
        stdMat.SetTextureScale("_MainTex", mat.GetTextureScale("baseColorTexture"));
        stdMat.SetTextureOffset("_MainTex", mat.GetTextureOffset("baseColorTexture"));
    }
}
`

## 5. VERTEX COLOR HANDLING

**Location:** C:\Users\flo\git\veldt\VeldtAR.Unity\Assets\Resources\ScriptableObjects\ModelData.cs

**Detection** (SetCuttingPlaneSubmeshMaterial method):
`csharp
bool hasVertexColors = false;
Mesh mesh = null;
if (renderer is SkinnedMeshRenderer smr)
    mesh = smr.sharedMesh;
else if (renderer is MeshRenderer)
{
    var mf = renderer.GetComponent<MeshFilter>();
    if (mf != null) mesh = mf.sharedMesh;
}
if (mesh != null)
    hasVertexColors = mesh.HasVertexAttribute(UnityEngine.Rendering.VertexAttribute.Color);
`

**Usage:** When cutting plane is active
`csharp
if (hasVertexColors)
    mat.EnableKeyword("_VERTEX_COLORS_ON");
`

**Limitation:** Vertex colors are ONLY used in CuttingPlane shader (_VERTEX_COLORS_ON keyword).
The Standard shader does NOT have vertex color support — vertex colors are ignored when Standard shader is applied.

## 7. CRITICAL ISSUES FOR WHITE RENDERING ON HOLOLENS

**Root Causes:**

### Issue A: Vertex Colors Not Supported in Standard Shader
- glTFast default uses vertex colors in some materials
- Material conversion (Phase 5) replaces all glTFast shaders with Standard
- Standard shader does NOT support vertex colors (COLOR_0 semantic)
- **Result:** CARTO animated GLB with vertex colors (COLOR_0) become white

**Evidence:**
- Loader.cs line 345: Skips metadata objects from conversion (preserves glTFast)
- Loader.cs lines 332-337: Explicitly checks for metadata to AVOID converting them
- This selective conversion suggests vertex color awareness

### Issue B: Texture Not Applied (Static GLB with UV-Baked Textures)
**Possible causes:**
1. baseColorTexture property not transferred if texture is null
2. Metallic/Roughness texture channels not handled (only scalars transferred, line 375)
3. Texture coordinate mismatch in conversion
4. glTFast material generator not binding textures correctly

**Evidence:**
- Line 364-372: Only transfers baseColorTexture, not metallicRoughnessTexture
- Line 375: Comment explicitly states "channel layout differs" for metallic/roughness

### Issue C: HoloLens Shader Stripping
- GLTFast uses shader_feature_local keywords
- Not referenced in scene at build time → stripped from builds
- Standard shader variants are pre-warmed (ShaderWarmup.cs lines 55-56, 212-220)
- But custom vertex color variants may not be warmed
