### KEY FINDINGS:

1. **RAW VOLTAGE DATA IS AVAILABLE** ✓
   - Located at: carto_mapper.py:513-559 (before colormap conversion)
   - Forms: LAT (milliseconds), bipolar voltage (mV), unipolar voltage (mV)
   - Type: numpy arrays [N] float64
   - Source: Either mesh file [VerticesColorsSection] OR sparse car-file points

2. **ALPHA CHANNEL IS CURRENTLY UNUSED** ✓
   - Always set to 1.0 (fully opaque)
   - Texture baking discards alpha (outputs RGB-only PNG)
   - No rendering impact since alphaMode=OPAQUE globally

3. **VOLTAGE → COLOR CONVERSION** ✓
   - Happens in: carto_colormaps.py:94-157 (_apply_colormap function)
   - Process: Normalize raw values to [0,1], interpolate through color stops
   - Color gradients: LAT (red→yellow→green→cyan→blue→purple)
   - Clamp ranges stored: (min,max) for each coloring type

4. **IMPLEMENTATION PATH CLEAR** ✓

### ARCHITECTURE OVERVIEW:

med2glb is a Python 3.10+ pipeline:
- Input: CARTO 3 .mesh + _car.txt files (and DICOM)
- Processing: Parse geometry, map sparse voltage measurements to mesh vertices
- Coloring: Apply LAT/bipolar/unipolar colormaps
- Export: GLB (glTF 2.0) with texture-baked vertex colors
- Target: HoloLens 2 AR visualization
